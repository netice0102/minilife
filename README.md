# minilife
极简生活网站
一、项目概述
1.1 基本信息
应用名称：MiniLife
网站类型：功能服务型
产品类型：工具型
核心价值：以极简列表形态帮助用户记录日常活动的时间消耗、事件内容、心情状态和类别，支持展开/收缩查看详细信息
目标用户：追求极简生活方式、希望高效管理时间的个人用户（25-45岁，注重生活品质和自我反思）
1.2 技术配置
纯静态语言网站不需要后台服务器，纯展示功能
数据库：不需要数据库（单条数据以json形式存储在单独目录）
网站主语言：中文
响应式要求：适配移动端/平板/桌面
2.1 功能模块
功能1：时间记录列表
以极简列表形态展示所有时间记录，包含时间、事件、用时、心情、类别五个核心字段。

使用场景：用户查看已记录的所有时间数据，快速浏览日常活动分布

操作流程：进入首页 → 查看列表 → 点击单条记录展开详情 → 再次点击收缩为极简状态

功能4：筛选与排序
用户可以根据类别、心情、时间范围等条件筛选记录，或按时间/用时排序。
使用场景：用户想查看特定类别的活动记录，或按时间顺序/用时长短整理记录
操作流程：点击筛选/排序按钮 → 选择筛选条件（类别/心情/时间范围）或排序方式 → 列表实时更新
优先级：中（增强）

三、交互与反馈
3.1 交互原则
核心原则：界面极简、操作直观、反馈清晰、无干扰

设计哲学：

列表默认收缩为极简状态，仅显示核心字段（时间、事件、用时、心情、类别）
点击单条记录展开详细信息，再次点击收缩
减少视觉噪音，聚焦核心信息
操作路径最短化，降低认知负担
3.2 状态反馈

短时加载：简约 loading 图标（旋转圆环）
列表加载：骨架屏占位

4.2 首页布局
整体布局模式：居中聚焦式布局

首屏即核心操作区，减少跳转，提升效率。

Hero区（首屏）：

背景：纯净浅色背景（米白/浅灰）
前景内容：
大标题："MiniLife"
副标题："记录时间，感知生活"
列表区域：

列表形态：垂直排列的卡片式列表
每条记录默认显示：时间 | 事件 | 用时 | 心情（图标）| 类别（标签）
点击展开：显示详细信息（可包含备注、创建时间等扩展字段）
再次点击：收缩回极简状态
筛选排序区：

位置：列表顶部或底部固定栏
筛选条件：类别下拉选择、心情下拉选择、时间范围选择
排序方式：按时间（最新/最早）、按用时（最长/最短）
五、视觉策略
5.1 品牌个性定位
核心定位：极简主义工具型产品

目标情感：用户使用工具时应该产生的核心感受

第一层感受：清晰 — 信息层级分明，无视觉干扰
第二层感受：平静 — 低饱和度配色，减少焦虑感
第三层感受：掌控 — 简洁的操作流程，高效完成任务
差异化策略：

我们不是复杂的时间管理软件，而是极简的生活记录工具
核心差异点：列表形态 + 展开/收缩交互 + 心情维度 + 克制的视觉设计
设计决策摘要：

主色：#F5F5F0（米白）| 辅色：#8C8C8C（中灰）| 强调色：#4A90A4（静谧蓝绿）
布局模式：居中聚焦式列表 + 卡片展开/收缩
动效基调：克制平滑 + 功能性反馈
字体策略：中等字号无衬线字体 + 充足行间距
采用趋势：精致微交互 + 无特殊装饰性趋势
5.2 设计语言
整体风格：极简生活风

核心调性：清晰、平静、克制

设计目标：通过低饱和度配色、充足留白、克制的动效，传递极简生活的平静感和对时间的掌控感。

5.3 色彩方案
主背景色：#F5F5F0（米白色）

应用：页面整体背景，传递柔和平静的视觉感受
主色调：#333333（深灰色）

应用：主要文字、标题，保证可读性同时避免纯黑的生硬感
辅助色：#8C8C8C（中灰色）

应用：次要文字、边框、分割线
强调色：#4A90A4（静谧蓝绿色）

应用：主要按钮、链接、选中状态，传递平静和专注感
功能性辅助色：

成功色：#6BBF8D（柔和绿色）
错误色：#E57373（柔和红色）
警告色：#FFB74D（柔和橙色）
心情图标配色：

开心：#FFD54F（暖黄）
平静：#4A90A4（蓝绿）
专注：#7E57C2（紫色）
疲惫：#90A4AE（灰蓝）
焦虑：#FF7043（橙红）
图片风格：不使用装饰性图片，仅使用简约线条图标和心情表情符号

5.4 布局与排版
布局原则：

居中聚焦：列表宽度控制在 600-800px（桌面端），移动端全宽
充足留白：卡片间距 16-24px，内边距 16-20px
清晰层级：通过字重、字号、颜色区分信息优先级
排版原则：

字体：无衬线字体（如 Inter、思源黑体）
标题字号：20-24px（桌面），18-20px（移动）
正文字号：14-16px
行间距：1.6-1.8 倍
按钮文字：直白清晰
5.5 交互动效
整体基调：克制平滑，功能性优先

核心动效：

列表项展开/收缩：高度平滑过渡 + 内容淡入淡出
按钮 hover：轻微背景色变化 + 位移 2px
筛选排序切换：列表项交错淡入（每项延迟 50ms）
动效参数规范：

过渡时长：展开/收缩 300ms，hover 效果 200ms，提交/删除 350ms
缓动函数：cubic-bezier(0.4, 0, 0.2, 1)
展开动画：高度 transition + opacity fade-in（内容区域）
成功/失败提示：持续时间 2 秒自动消失，配合图标缩放（0→1，200ms）
hover 效果：背景色变化（透明度 0.05→0.1），位移 2px 向上
交错延迟：列表刷新时每项间隔 50ms
禁止：

❌ 避免弹跳、抖动等活泼动画
❌ 避免过多装饰性动效，影响阅读和操作效率
✅ 所有动效必须服务于功能反馈和信息层级
六、多端适配策略
6.1 响应式要求
确保所有设备上列表清晰可读、操作便捷。

6.2 断点定义
设备类型	屏幕宽度	优化重点
移动端	< 768px	列表全宽，大按钮，简化筛选栏
平板	768px - 1023px	列表宽度 700px，横屏优化
桌面端	≥ 1024px	列表宽度 800px，完整功能展示
6.3 关键组件适配
列表区域：

移动端：全宽布局，卡片内边距 12-16px，字体略小（14px）
桌面端：居中 800px 宽度，卡片内边距 20px，字体 16px

移动端：底部固定悬浮按钮（FAB）或顶部固定栏
桌面端：顶部固定栏或列表右上角
筛选排序栏：

移动端：折叠为下拉菜单，点击展开
桌面端：水平排列，常驻显示
性能优化：

移动端：简化动效，列表虚拟滚动（记录较多时）
桌面端：完整交互动效
